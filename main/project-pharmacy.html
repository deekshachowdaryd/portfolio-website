Pharmacy Database Management System

A desktop database management application designed to streamline pharmacy operations. The system manages medicine records, tracks real-time inventory, and handles basic billing operations through a GUI client.

Tech Stack

Python Tkinter MySQL python-dotenv

Key Features

  • Desktop Interface: Intuitive GUI built with Tkinter for staff and administrators to interact with medicine records.
  • Inventory Management: Track manufacturing dates, expiration dates, lot numbers, manufacturer information, and pricing for all inventory.
  • Persistent SQL Database: Powered by MySQL with direct connections managed via mysql-connector.
  • CRUD Operations: Full capability to add new medicines, update existing records, search current stock, and remove records directly through the interface.
  • Tabular View: Interactive ttk.Treeview displaying real-time database query results with functional scrollbar adjustments.
  • Secure Configuration: Utilizes python-dotenv to keep sensitive database credentials out of code repositories.

Challenges

One of the primary challenges was synchronizing the GUI state in Tkinter with the active relational tables in MySQL. Ensuring input validations (such as preventing duplicate primary key Reference Numbers and verifying positive integer values for stock units) required robust validation layers and clear error dialog integration.

What I Learned

This project enhanced my understanding of GUI-to-database integration. I learned how to connect Python desktops to SQL servers, design functional tables for relational databases, build database transaction layers, and manage application configurations using environmental variables.
